# Security Policy

## Supported Versions

We release patches for security vulnerabilities in these versions of our projects:

| Version | Supported          |
| ------- | ----------------- |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please send an email to ulisescf.24@gmail.com instead of opening a GitHub issue. We will acknowledge your report within 24 hours and provide an estimated timeline for a fix.

Please include the following information in your report:
- Type of vulnerability
- Full paths of source file(s) related to the vulnerability
- Step(s) to reproduce the issue
- Proof-of-concept or exploit code (if possible)

We request that you:
- Give us reasonable time to address the vulnerability before disclosing it publicly
- Do not exploit the vulnerability for any reason

## Security Updates

Security updates will be released as patch versions and listed in the CHANGELOG.
